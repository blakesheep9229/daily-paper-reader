## 摘要
大型语言模型（LLM）在电子商务社区推荐中展现了良好的潜力。尽管 LLM 和多模态大型语言模型（MLLM）通常用于将笔记编码为隐式嵌入，但利用其生成能力以可解释的标签来表示笔记的研究仍然鲜见。在标签生成领域，传统的封闭式方法高度依赖于标签库的设计，而现有直接应用于笔记推荐的开放式方法面临两个局限：（1）MLLM 在生成过程中缺乏指导，产生的标签冗余且未能准确捕捉用户兴趣；（2）生成的标签通常较为粗糙，无法充分细粒度地表示笔记内容，从而干扰后续推荐效果。为了解决这些问题，我们提出了 TagLLM——一种用于笔记推荐的细粒度标签生成方法。TagLLM 通过“用户兴趣手册”来捕捉用户在笔记类别上的兴趣，并利用多模态 CoT 提取构建细粒度标签数据。同时，我们设计了标签知识蒸馏方法，使小模型具备具有竞争力的生成能力，从而提升推理效率。在在线 A/B 测试中，TagLLM 使单个用户的平均观看时长提升 0.31%，平均交互次数提升 0.96%，并在冷启动场景下将页面浏览点击率提高 32.37%，验证了其有效性。

## Abstract
Large Language Models (LLMs) have shown promising potential in E-commerce community recommendation. While LLMs and Multimodal LLMs (MLLMs) are widely used to encode notes into implicit embeddings, leveraging their generative capabilities to represent notes with interpretable tags remains unexplored. In the field of tag generation, traditional close-ended methods heavily rely on the design of tag pools, while existing open-ended methods applied directly to note recommendations face two limitations: (1) MLLMs lack guidance during generation, resulting in redundant tags that fail to capture user interests; (2) The generated tags are often coarse and lack fine-grained representation of notes, interfering with downstream recommendations. To address these limitations, we propose TagLLM, a fine-grained tag generation method for note recommendation. TagLLM captures user interests across note categories through a User Interest Handbook and constructs fine-grained tag data using multimodal CoT Extraction. A Tag Knowledge Distillation method is developed to equip small models with competitive generation capabilities, enhancing inference efficiency. In online A/B test, TagLLM increases average view duration per user by 0.31%, average interactions per user by 0.96%, and page view click-through rate in cold-start scenario by 32.37%, demonstrating its effectiveness.

## 二、方法论：TagLLM 框架

TagLLM 是一个多阶段的 **细粒度标签生成与蒸馏框架**，核心思想是在多模态输入上引入兴趣引导的生成链路。

### (1) 总体流程
由四个阶段组成（见图 2）：
1. **用户兴趣手册（User Interest Handbook）生成**  
   - LLM 先根据笔记类别（如服饰、游戏、数码等）自动扩展出可能的兴趣维度（如“颜色”“品牌”“使用场景”等）。  
   - 人工专家根据平台目标对这些维度进行精修，形成用于指导生成的“手册”。  

2. **标签数据构建（Tag Data Construction）**  
   - 使用 **多模态输入**（图像、文本、ASR 音频转录）融合。
   - 采用 **CoT（Chain-of-Thought）抽取** 三步法：
     1. *Handbook-based Generate*：按手册维度生成初始标签。
     2. *Low-info Merge*：合并低信息或冗余标签。
     3. *Importance Rank*：为每个标签打分（1–5）以反映核心内容。  
   - 输出高质量、聚焦的训练数据。

3. **标签知识蒸馏（Tag Knowledge Distillation）**  
   - 为减少推理成本，将大模型生成能力蒸馏至小模型。  
   - 包含两阶段：
     - **SFT（Supervised Fine-Tuning）**：标准监督训练建立初始生成能力。  
     - **DPO（Direct Preference Optimization）**：基于偏好对比强化模型。  
   - 评估采用 **MLLM-as-a-judge** 策略（以 Doubao-1.6-thinking 作为裁判），通过“真实性、覆盖性、重要性”三维度打分。  

4. **推荐特征生成（Recommendation Feature）**  
   - 将生成的标签用于：
     - **用户特征**：根据历史交互为标签打分形成“用户画像”；
     - **项目特征**：将标签序列编码成向量，作为推荐输入。  
   - 最终用于在线双塔推荐系统的训练与推理。

### (2) 算法结构说明
- 模型采用多模态输入 (图、文、音)，CoT 过程借助提示模板分步执行。  
- 蒸馏优化公式主要包括：
  - **BCE 损失**（用于点击建模）；
  - **对比学习损失 CL**（增强标签-用户-物品之间的匹配语义）；
  - 最终优化目标：
    \[
    L = L_{BCE} + \lambda_1 L_{CL} + \lambda_2 ||\Theta||^2
    \]
  - 其中 \(\lambda_1, \lambda_2\) 为权重，用于平衡正则与对比目标。
