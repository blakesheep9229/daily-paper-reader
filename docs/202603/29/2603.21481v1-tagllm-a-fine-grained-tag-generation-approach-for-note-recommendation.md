## 摘要
大型语言模型（Large Language Models, LLMs）在电子商务社区推荐中展现出良好的潜力。虽然 LLMs 与多模态大型语言模型（Multimodal LLMs, MLLMs）广泛用于将笔记编码为隐式嵌入，但利用其生成能力以可解释标签的形式表征笔记的研究仍然缺乏。在标签生成领域，传统的封闭式方法高度依赖于标签库的设计，而现有直接应用于笔记推荐的开放式方法存在两个主要问题：（1）MLLMs 在生成过程中缺乏指导，导致产生的标签冗余且无法准确反映用户兴趣；（2）生成的标签往往过于粗糙，缺乏对笔记内容的精细表征，从而干扰下游推荐效果。为解决这些问题，我们提出了 TagLLM，这是一种面向笔记推荐的细粒度标签生成方法。TagLLM 通过用户兴趣手册（User Interest Handbook）在不同笔记类别中捕捉用户兴趣，并利用多模态链式思维提取（multimodal CoT Extraction）构建细粒度标签数据。此外，我们提出了标签知识蒸馏（Tag Knowledge Distillation）方法，使小模型也具备具有竞争力的生成能力，从而提升推理效率。线上 A/B 测试结果表明，TagLLM 使用户平均观看时长提升了 0.31%，用户平均互动次数提升了 0.96%，在冷启动场景下的页面浏览点击率提升了 32.37%，验证了其有效性。

## Abstract
Large Language Models (LLMs) have shown promising potential in E-commerce community recommendation. While LLMs and Multimodal LLMs (MLLMs) are widely used to encode notes into implicit embeddings, leveraging their generative capabilities to represent notes with interpretable tags remains unexplored. In the field of tag generation, traditional close-ended methods heavily rely on the design of tag pools, while existing open-ended methods applied directly to note recommendations face two limitations: (1) MLLMs lack guidance during generation, resulting in redundant tags that fail to capture user interests; (2) The generated tags are often coarse and lack fine-grained representation of notes, interfering with downstream recommendations. To address these limitations, we propose TagLLM, a fine-grained tag generation method for note recommendation. TagLLM captures user interests across note categories through a User Interest Handbook and constructs fine-grained tag data using multimodal CoT Extraction. A Tag Knowledge Distillation method is developed to equip small models with competitive generation capabilities, enhancing inference efficiency. In online A/B test, TagLLM increases average view duration per user by 0.31%, average interactions per user by 0.96%, and page view click-through rate in cold-start scenario by 32.37%, demonstrating its effectiveness.

## 二、方法论与关键技术细节

TagLLM 由四个核心模块构成（见 Figure 2）：

### 1. **User Interest Handbook 生成**
- 针对每个笔记类别（如服饰、美妆、游戏）自动构建“用户兴趣手册”，定义该类别下的主要兴趣维度（如品牌、材质、风格、季节、场景等）。
- 实现过程：
  - 使用 LLM 自动扩展潜在兴趣点；
  - 由人工专家审核、精炼，形成最终手册；
  - 每个类别对应不同维度集合，体现用户关注差异。

### 2. **Tag 数据构建（多模态 Chain-of-Thought 提取）**
- 输入：笔记中的多模态信息（文本、图片、语音转录）。
- 三步 CoT（Chain-of-Thought）流程：
  1. **Handbook-based Generate**：依据手册维度指导生成候选标签；
  2. **Low-Info Merge**：合并无信息或冗余的标签；
  3. **Importance Rank**：为每个标签打分（1–5）以区分核心与次要内容。
- 输出：包含权重的细粒度、多模态标签数据。

### 3. **Tag Knowledge Distillation**
- 目标：在保证生成质量的同时提升小模型推理效率。  
- 过程分为两阶段：
  - **SFT（监督微调）阶段**：使用 SFT 数据集进行标准的下一词预测训练；
  - **DPO（Direct Preference Optimization）阶段**：用基于 Pairwise Comparison（成对比较）生成的偏好样本进行优化。  
- **评估框架**：  
  - 采用 “MLLM-as-a-judge” 自动打分体系，维度为 *真实性（Truth）*、*覆盖度（Coverage）* 和 *重要性（Importance）*；  
  - 使用大模型（如 doubao-1.6-thinking）作为“评审模型”，结合人工验证保证客观性。

### 4. **推荐特征转换**
- 将生成的标签转换为两类特征：
  - **User Profile**：基于用户交互行为（点击、点赞、收藏）构造兴趣标签画像；
  - **Tag Embedding**：将笔记对应的细粒度标签输入预训练编码器，映射到潜空间，用于Note特征表达。
- 最终嵌入与用户画像在**双塔结构（User Tower / Item Tower）**中联合优化，通过**BCE + 对比学习（CL）损失**进行训练。

## 三、实验设计

### 1. **数据集**
- 来源：得物（Dewu）平台的真实电商社区笔记。  
- 包含 18 个类别，如鞋履、服饰、美妆、家居、数码、游戏等。

### 2. **基线模型**
- 多个小规模 MLLM：
  - InternVL3-2B / 3.5-2B / 3.5-4B；
  - Qwen3-VL-2B / Qwen3-VL-4B；
- 用于评估蒸馏效果与性能对比。

### 3. **评估指标**
- **Basic Attribute Score（BAS）**：三个维度真值判断；
- **Pairwise Score（PS）**：基于对比优劣的可微量化评分；
- **线上指标**：AVDU（观看时长）、AIU（互动次数）、PVCTR（点击率）、PVIR（互动率）等。

## 六、主要结论与发现

- TagLLM 能有效生成聚焦、精细、符合用户兴趣的多模态标签；
- 经过两阶段知识蒸馏后，小模型性能与大模型相近；

  - 冷启动点击率提升 **32.37%**；
  - 冷启动互动率提升 **46.91%**；
- 在冷启动与低交互场景中尤为显著，表明细粒度标签能改善推荐覆盖与解释性。
